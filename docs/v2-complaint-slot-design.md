# MedAsk V2 四主诉槽位与识别规则草案

> 设计状态：Phase 1，仅供信息整理模块设计与合成评测。不得直接视为生产规则或完整分诊标准。

## 1. 产品范围

### 目标用户

18—65 岁、希望在沟通前整理当前症状信息的成人用户。

### 核心任务

用户用自然语言描述头痛、头晕、腹痛或胸部不适后，系统在最多 7 轮内整理为可核对的结构化摘要；系统不推断疾病，不生成诊断、药物、剂量或治疗建议。

### 最短完整流程

1. 校验年龄和非支持人群。
2. 对用户原文执行确定性风险检查；命中当前肯定风险表达立即停止普通流程，Provider 调用次数为 0。
3. 识别一个或多个症状主诉，并区分当前、否定、历史、已缓解、假设和不确定。
4. 从首句提取有逐字证据的槽位候选；模型只能提出候选。
5. 由确定性 Schema、白名单、上下文、冲突和接受策略决定是否写入。
6. 按跨主诉共享和单主诉优先级选择最多 7 个问题，允许用户跳过。
7. 输出仅由用户信息组成的摘要，明确未回答、已跳过和不适用字段。

### V1 范围、后续范围与非目标

- 第一批 V1：`headache`、`dizziness` 的确定性识别、槽位、Mock 和合成评测。
- 第二批：完成定向标注后再评审 `abdominal_pain`。
- 暂缓：`chest_discomfort` 普通流程接入；只保留现有风险中断能力。
- 非目标：科室选择、疾病推断、诊断、药物、剂量、治疗建议、真实 Provider Prompt 修改、儿科、孕产妇、肿瘤治疗和复杂确诊随访。

## 2. 通用槽位原则

### 共享槽位

| 槽位 ID | 字段 | 类型 | 默认级别 | 摘要区 | 说明 |
|---|---|---|---|---|---|
| `onset` | 开始时间 | text | 必问 | onset | 记录用户原话或安全规范化时间，不推断病程 |
| `medicationHistory` | 已采取措施 | text | 必问 | measures | 只记录已做过什么；不得反向生成建议；允许跳过 |
| `otherCurrentComplaints` | 同时存在的其他当前主诉 | string[] | 可选 | associated | 只允许受支持主诉 ID；多主诉不强制选一个“主要疾病” |

### 7 轮调度规则

1. 年龄、人群和风险校验不计入普通问答轮次。
2. 首句已经通过确定性验证的槽位不再重复询问。
3. 多主诉只共享 `onset` 和 `medicationHistory`；规律与功能影响保留主诉专属字段，避免不同体验互相覆盖。
4. 每个主诉先问必问槽位，再问能改变摘要完整度的条件槽位。
5. 风险相关布尔值不得由模型自动写入；用户在结构化问题中明确确认时，仍由确定性风险规则决定中断。
6. 到达 7 轮时生成摘要并列出未回答字段，不用模型补齐。
7. 所有槽位都提供“暂不清楚/跳过”；必问表示优先询问，不表示强迫作答。

### 候选接受边界

- 候选至少包含 `fieldId`、`value`、`confidence`、`evidence` 和 `status`。
- `status` 只允许 asserted、negated、uncertain、historical、resolved、hypothetical。
- evidence 必须逐字存在于本轮完整原文，并在局部上下文支持候选状态。
- 只有 asserted、字段白名单内、类型合法、无风险且不冲突的候选可进入接受策略。
- 同值为 no-op；不同值生成固定澄清，不覆盖旧值。
- 诊断、疾病概率、科室、药物、剂量、治疗、风险等级和最终业务结果均为禁止字段。

## 3. `headache` 槽位草案

### 用户目标

整理头部疼痛何时开始、如何出现、位于何处、体验如何、如何变化、影响多大以及伴随的非风险信息。

### 槽位

| 类型 | 槽位 ID | 问题含义 | 可跳过 | 优先级 |
|---|---|---|---|---:|
| 必问 | `onset` | 什么时候开始 | 是 | 10 |
| 必问 | `headacheOnsetSpeed` | 突然出现、逐渐出现或不清楚 | 是 | 11 |
| 必问 | `headacheFunctionalImpact` | 对工作、睡眠或日常活动的影响 | 是 | 20 |
| 必问 | `headacheLocation` | 前额、太阳穴、后脑勺、单侧、全头或不清楚 | 是 | 30 |
| 必问 | `headachePattern` | 持续、间歇、反复或不清楚 | 是 | 40 |
| 可选 | `headacheSensation` | 疼痛体验，仅记录用户原话 | 是 | 50 |
| 可选 | `headacheEpisodeDuration` | 单次大约多久 | 是 | 60 |
| 共享 | `medicationHistory` | 已采取过什么措施 | 是 | 70 |

### 最多 7 轮优先级

`onset` → `headacheOnsetSpeed` → `headacheFunctionalImpact` → `headacheLocation` → `headachePattern` → `medicationHistory` → `headacheSensation`。已由首句填充的槽位跳过；有剩余轮次时再收集 `headacheEpisodeDuration`。

### 摘要展示字段

主诉名称、开始时间、出现方式、位置、发作规律、单次持续时间、体验描述、功能影响、用户明确提供的伴随信息、已采取措施、未回答和跳过项。

### 不适用条件

- 仅描述头皮、头发、面部、牙齿、颈部或外伤表面局部疼痛，且没有头痛表达。
- 只谈远期既往头痛、明确否定或未来担忧；近期已缓解可进入本次整理，但只显示在“本次已缓解”。
- 年龄不在 18—65 岁，或属于孕产妇、复杂确诊随访等非支持范围。
- 命中当前肯定风险表达时不进入普通槽位流程。

## 4. `dizziness` 槽位草案

### 用户目标

整理“晕”的主观体验、开始时间、持续与诱发方式、平衡和活动影响以及同时出现的非风险信息。

### 槽位

| 类型 | 槽位 ID | 问题含义 | 可跳过 | 优先级 |
|---|---|---|---|---:|
| 必问 | `onset` | 什么时候开始 | 是 | 10 |
| 必问 | `dizzinessExperience` | 天旋地转、发飘/发昏、像要晕倒、站不稳或不清楚 | 是 | 15 |
| 必问 | `dizzinessFunctionalImpact` | 对站立、行走、工作或日常活动的影响 | 是 | 20 |
| 必问 | `dizzinessPattern` | 持续、间歇、反复或不清楚 | 是 | 30 |
| 必问 | `dizzinessTrigger` | 起身、转头、活动或其他/不清楚 | 是 | 40 |
| 可选 | `balanceImpact` | 是否可独立行走、走路发飘或需要扶靠 | 是 | 45 |
| 可选 | `dizzinessEpisodeDuration` | 每次大约多久 | 是 | 55 |
| 共享 | `medicationHistory` | 已采取过什么措施 | 是 | 70 |

### 最多 7 轮优先级

`onset` → `dizzinessExperience` → `dizzinessFunctionalImpact` → `dizzinessPattern` → `dizzinessTrigger` → `medicationHistory` → `balanceImpact`。首句已填字段跳过；有剩余轮次时再收集 `dizzinessEpisodeDuration`。

### 摘要展示字段

主诉名称、开始时间、晕的体验、发作规律、单次持续时间、诱发方式、活动/平衡影响、用户明确提供的伴随信息、已采取措施、未回答和跳过项。

### 不适用条件

- 仅表示困倦、疲劳、“看视频看晕了”、晕车或情绪性口语，且没有当前头晕体验。
- 仅描述远期历史、明确否定或假设；近期已缓解可进入本次整理，但只显示在“本次已缓解”。
- 年龄或人群不支持，或属于复杂确诊随访。
- 命中晕厥、突然语言/视力/肢体变化、严重呼吸困难等当前风险表达。

## 5. `abdominal_pain` 槽位草案

### 用户目标

整理腹部疼痛的位置、开始时间、变化、体验、功能影响，以及排便、呕吐、进食或排尿相关的用户原始信息。

### 槽位

| 类型 | 槽位 ID | 问题含义 | 可跳过 | 优先级 |
|---|---|---|---|---:|
| 必问 | `onset` | 什么时候开始 | 是 | 10 |
| 必问 | `abdominalLocation` | 上腹、下腹、左侧、右侧、肚脐周围、全腹或不清楚 | 是 | 15 |
| 必问 | `episodePattern` | 持续、间歇、反复或不清楚 | 是 | 20 |
| 必问 | `severityImpact` | 对活动、睡眠或进食的影响 | 是 | 25 |
| 可选 | `abdominalCharacter` | 胀、绞、刺、灼、隐痛或用户原话 | 是 | 35 |
| 可选 | `bowelChange` | 腹泻、便秘、排便变化或无明确变化 | 是 | 40 |
| 可选 | `vomitingPresent` | 是否呕吐；呕血表达由风险规则抢占 | 是 | 45 |
| 条件 | `foodRelation` | 用户明确表示与进食前后有关时记录 | 是 | 55 |
| 条件 | `urinaryChange` | 用户明确表示排尿异常时记录 | 是 | 60 |
| 共享 | `measuresTaken` | 已采取过什么措施 | 是 | 90 |

### 最多 7 轮优先级

`onset` → `abdominalLocation` → `episodePattern` → `severityImpact` → `abdominalCharacter` → `bowelChange` → `vomitingPresent`。`foodRelation`、`urinaryChange` 和 `measuresTaken` 仅在首句已有或有剩余轮次时收集。

### 摘要展示字段

主诉名称、开始时间、位置、规律、体验、功能影响、排便变化、呕吐、进食/排尿相关信息、已采取措施、未回答和跳过项。

### 不适用条件

- 仅描述腹壁肌肉、肚皮伤口、腰背或肋骨局部疼痛，且没有腹部内部疼痛表达。
- 只有“胃不舒服”“肚子怪怪的”等非疼痛模糊表达时先澄清，不直接写入腹痛。
- 孕产妇和年龄不支持人群不进入普通流程。
- 突发剧烈腹痛、呕血/便血或黑便、明显腹胀且无法排便排气、晕厥等表达先中断。

## 6. `chest_discomfort` 槽位草案

### 用户目标

整理胸部或呼吸不适的体验、位置、开始和持续方式、活动关系、功能影响及同时出现的信息；任何当前明确胸痛或严重呼吸困难仍由风险规则先行中断。

### 槽位

| 类型 | 槽位 ID | 问题含义 | 可跳过 | 优先级 |
|---|---|---|---|---:|
| 必问 | `onset` | 什么时候开始 | 是 | 10 |
| 必问 | `chestSensation` | 胸闷、发紧、疼痛、气短、呼吸不畅或不清楚 | 是 | 15 |
| 必问 | `episodePattern` | 持续、间歇、反复或不清楚 | 是 | 20 |
| 必问 | `severityImpact` | 对说话、走动或日常活动的影响 | 是 | 25 |
| 可选 | `chestLocation` | 胸前、左侧、右侧、中央、弥漫或不清楚 | 是 | 30 |
| 可选 | `chestEpisodeDuration` | 每次大约多久 | 是 | 35 |
| 可选 | `activityRelation` | 活动时、静息时、两者都有或不清楚 | 是 | 40 |
| 条件 | `breathingImpact` | 出现呼吸不适时记录是否影响完整说话或轻微活动；严重表达先中断 | 是 | 45 |
| 可选 | `chestNonRiskAssociated` | 出汗、恶心、头晕等用户明确表达的信息；组合风险先中断 | 是 | 50 |
| 共享 | `measuresTaken` | 已采取过什么措施 | 是 | 90 |

### 最多 7 轮优先级

只有在未命中风险规则且未来完成独立安全评审后，才考虑：`onset` → `chestSensation` → `episodePattern` → `severityImpact` → `chestEpisodeDuration` → `activityRelation` → `breathingImpact`。本阶段不接入生产普通流程。

### 摘要展示字段

主诉名称、开始时间、胸部/呼吸体验、位置、规律、单次持续时间、活动关系、功能影响、用户明确提供的伴随信息、已采取措施、未回答和跳过项。

### 不适用条件

- 单纯心慌、心跳快、心悸，而没有胸部或呼吸不适表达。
- 仅描述乳房、胸壁伤口、肋骨外伤处或皮肤表面不适时先澄清；当前明确胸痛仍按保守风险规则处理。
- 历史、已缓解、明确否定或假设表达不进入当前普通流程。
- 当前胸痛、持续压迫/紧缩、放射样描述、明显呼吸困难、无法完整说话、晕厥等先中断。

## 7. 主诉识别规则草案

识别顺序：局部上下文切分 → 否定/历史/已缓解/假设判定 → 当前肯定主诉匹配 → 多主诉合并 → 模糊或冲突澄清。不得仅靠全句是否出现关键词。

### 7.1 `headache`

- 正向关键词：头痛、头疼。
- 常见口语：脑袋疼、太阳穴疼、后脑勺疼、前额疼、半边头疼、整个头都疼。
- 否定表达：没有头痛、不头疼、否认头痛、未出现头痛、不伴头痛。
- 局部误识别：头皮疼、头发根疼、额头磕伤处疼、牙疼牵着脸疼、脖子疼；没有明确头部内部疼痛时不自动匹配。
- 历史/已缓解：去年头痛、以前常头疼、之前头痛现在好了、头疼已经消失。
- 多主诉：头痛并头晕；发热同时头痛；头痛加胸闷；分别保留当前肯定标签。
- 易混淆：头重、脑子发木、眼睛疼、脸疼、“偏头痛”作为既往确诊随访；模糊时澄清，不推断。

### 7.2 `dizziness`

- 正向关键词：头晕、头昏、眩晕、发晕。
- 常见口语：晕乎乎、天旋地转、站起来发晕、感觉要晕、走路发飘、站不稳。
- 否定表达：没有头晕、不晕、否认眩晕、未出现发晕、走路不晃。
- 局部误识别：晕车、看屏幕看晕、困得发晕、被消息“整晕了”、单纯眼花；需要确认是否为当前身体体验。
- 历史/已缓解：以前头晕、上个月眩晕、刚才有点晕现在好了、头晕已经消失。
- 多主诉：头晕并头痛；头晕加胸闷；腹痛后又头晕；保留各自主诉和独立上下文。
- 易混淆：疲倦、嗜睡、意识不清、晕厥、平衡差；晕厥或意识异常走风险流程，其他模糊表达先澄清。

### 7.3 `abdominal_pain`

- 正向关键词：腹痛、肚子疼、腹部疼痛。
- 常见口语：胃疼、上腹疼、下腹疼、小腹疼、肚脐周围疼、左边/右边肚子疼。
- 否定表达：没有腹痛、肚子不疼、否认胃疼、不伴腹部疼痛。
- 局部误识别：腹肌酸痛、肚皮伤口疼、腰疼、肋骨下表面疼、裤腰勒得疼；没有腹部内部疼痛表达时不自动匹配。
- 历史/已缓解：去年腹痛、之前胃疼、刚才肚子疼现在好了、腹痛已经缓解。
- 多主诉：腹痛并头晕；腹痛加胸闷；腹痛同时发热；分别判断当前性和风险。
- 易混淆：胃不舒服、腹胀、反酸、恶心、想吐、腹泻但未表达疼痛；先澄清是否存在疼痛。

### 7.4 `chest_discomfort`

- 正向关键词：胸闷、气短、呼吸困难、呼吸不畅、胸痛、胸口疼。
- 常见口语：胸口发闷、胸口发紧、喘不上气、气不够用、上气不接下气、呼吸很费力。
- 否定表达：没有胸闷、不气短、否认胸痛、呼吸不困难、不觉得胸口紧。
- 局部误识别：单纯心慌/心悸、乳房疼、胸壁伤口疼、肋骨按压疼、嗓子堵；需要局部上下文和澄清。
- 历史/已缓解：去年胸闷、以前气短、刚才胸口紧现在好了、呼吸不适已经消失。
- 多主诉：胸闷并头晕；胸痛加气短；腹痛同时胸闷；先执行风险组合规则，再决定是否进入普通流程。
- 易混淆：单纯心跳快、焦虑描述、胃部烧灼感、运动后正常喘、鼻塞；不得推断原因，模糊时澄清。

## 8. 成功标准与失败回退

- 成功：在最多 7 轮内生成只含用户已确认信息的摘要；未回答和跳过项可见。
- 识别模糊：保持原状态并询问固定澄清问题。
- 槽位冲突：显示旧值与新候选，要求用户确认；不自动覆盖。
- 风险命中：停止普通流程，不调用模型，不推断疾病。
- Provider、Schema 或 evidence 失败：回退标准问题，状态和轮次不因失败改变。
- 不支持人群或意图：进入 `unsupported`，不尝试通过模型绕过范围。

## 9. 停止条件

- 四类槽位、优先级、摘要字段、不适用条件和识别规则均已形成草案。
- 第一批范围固定为 `headache` + `dizziness`，未扩大科室或人群。
- 所有风险字段明确由确定性代码控制，模型没有直接写入权。
- 本轮不把草案写入生产 `complaintRules`、类型或页面入口。
