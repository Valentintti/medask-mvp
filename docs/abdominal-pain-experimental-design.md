# MedAsk 腹痛实验性规则模块

> 状态：`experimental`。本模块只整理18—65岁成人自行提供的腹痛语言信息，不提供疾病诊断、概率、检查、药物、剂量或治疗建议。

## 1. 发布决策与证据边界

腹痛人工数据门禁未完全通过，既有失败报告继续保留，未被删除或改写。受项目时间约束，本轮不再开展额外人工裁决，而是进入实验性规则实现：采用保守识别、固定澄清、儿童孕产妇分流和风险前置来降低误用。

现有AI暂定标签、20条复核、8条补充以及合成案例仅作为设计与回归依据，不视为人工金标。本模块的测试指标属于合成语言工程指标，不代表腹痛数据门禁通过，更不是临床准确率或完整分诊标准。

发布范围明确分离：

- GitHub Pages 规则版：实验性支持 `abdominal_pain`；
- 开发 Mock：实验性支持 `abdominal_pain`；
- 真实 Provider：`abdominal_pain` 保持 disabled，服务端白名单仍只有已验证的 `fever/cough`；
- `chest_discomfort`：不开发、不提供入口。

## 2. 主诉和当前性边界

规则可识别明确腹部、肚子或胃部疼痛，包括腹痛、肚子疼/痛、上下腹痛、左右上下腹痛、胃疼/痛、肚脐周围疼和小腹疼。腰痛、后背痛、胸痛、头痛，以及单纯腹胀、恶心、反酸或胃口差不映射为腹痛。

否定、纯历史、假设和疾病知识表达不进入当前流程。近期本次腹痛已缓解可以进入信息整理，主诉状态记为 `resolved`，摘要显示“本次腹痛目前已缓解”，不显示为当前仍腹痛。模糊的“胃还是肚子说不清”“腰腹部不舒服”“好像肚子有点不适”不自动写入腹痛，先问固定问题：“你现在是否明确有腹部、肚子或胃部疼痛？”该确认不占普通7轮。

明显表单占位、医生回答、治疗建议、SEO内容或多段医疗文本拼接会被拒绝。产品输入没有 title/ask 双字段；在上游数据准备中，ask为空时是否回退title仍由数据治理项目记录来源，本模块不会把医生answer作为患者症状。

## 3. 七轮槽位

普通腹痛流程最多7轮，顺序为：

1. `onset`：什么时候开始；
2. `abdominalLocation`：上腹、下腹、左侧、右侧、肚脐周围、位置不清；
3. `abdominalPattern`：持续、阵发、反复、不确定；
4. `abdominalFunctionalImpact`：对活动、睡眠或进食的影响；
5. `abdominalSensation`：用户自己的疼痛描述；
6. `abdominalAssociatedStatus`：呕吐、排便变化、均没有、不确定；
7. `medicationHistory`：只记录已经采取的措施。

所有字段允许跳过。`onset` 和 `medicationHistory` 与其他主诉按 slotId 合并；最后一轮为“已采取措施”预留，不突破7轮，也不提供任何处理建议。

## 4. 风险前置

以下当前肯定表达在任何 Mock/Provider 调用前停止普通流程：突然剧烈腹痛、呕血、明显便血或黑便、明显腹胀且无法排便排气、腹痛伴晕厥/意识异常、腹痛伴严重呼吸困难。固定提示只要求人工或线下判断，不推断疾病。

否定、历史、已缓解和假设语境不升级。“有点疼”“疼了几天”不等于突然剧烈；只有排便困难而没有明确腹胀及无法排气也不触发该组合规则。咯血、血尿和明确胸痛由全局风险规则处理，不计为腹痛特异规则。

## 5. 合成工程评测

评测入口：`pnpm eval:abdominal:experimental`。现有33条腹痛合成设计案例参与边界回归，另有13项补充合成检查。当前聚合结果为：主诉识别38/38、当前性38/38、风险前置4/4、不支持人群2/2；相邻非腹痛假阳性、非当前误写、否定风险误触发、共享槽位重复和摘要编造均为0。

冲突类案例中的孤立追问片段依赖已有会话上下文，不进入首句识别分母。上述结果不能外推到真实患者语言，也不能用来声称临床安全或医疗有效性。
